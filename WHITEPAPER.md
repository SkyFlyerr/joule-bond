## 1. Abstract

Joule Bond introduces **eJLE** (eJoule Energy), a Real World Asset (RWA) token where each unit represents **1 kWh (kilowatt-hour)** of verified energy generation capacity. Unlike volatile cryptocurrencies, eJLE is a digital wrapper around real physical work — whether generated by a nuclear plant, a solar panel, or human labor.

```
1 eJLE = 1 kWh = 1,000 kJ = 3.6 MJ
```

The protocol creates a **universal unit of account** for all forms of energy and productive work, enabling a fully decentralized marketplace where anyone can mint, trade, and redeem tokens backed by their validated productive capacity.

**Key Innovations:**
- **100% Collateral-Backed**: Every eJLE in circulation is backed by staked crypto assets
- **Sponsored Minting**: Stakers enable newcomers to participate without capital requirements
- **4-Tier AI-Enhanced Arbitration**: Combining AI, crowdsourced jurors, and professional arbitrators
- **DAO Governance**: Fully decentralized protocol control with stake-weighted voting

## 2. Introduction

### 2.1 The Problem

The global energy market suffers from fundamental inefficiencies:

- **Fragmented Trading**: Energy is traded through opaque, centralized intermediaries
- **No Universal Standard**: Different regions use incompatible pricing and measurement systems
- **Limited Access**: Small generators (rooftop solar, individuals) cannot efficiently monetize their capacity
- **Value Abstraction**: Traditional money is disconnected from physical reality

### 2.2 The Vision

> "Everything in the universe is energy. Money should reflect that."

Joule Bond proposes a paradigm shift:

- Power plants sell electricity as tokens
- Solar panels monetize their output directly
- **Humans sell their labor measured in energy units**
- All goods and services priced in kWh

### 2.3 Core Principles

| Principle | Description |
|-----------|-------------|
| **Physical Backing** | Every token represents real productive capacity |
| **Universal Access** | Any validated generator can participate |
| **Decentralized Control** | DAO governance with no admin keys |
| **Fair Arbitration** | AI-enhanced multi-tier dispute resolution |
| **Sustainable Economics** | Burn-to-mint mechanism prevents inflation |

## 4. Token Economics

### 4.1 The Mint-Sell-Burn Cycle

```
┌─────────────────────────────────────────────────────────────────┐
│                    THE eJLE CYCLE                                │
│                                                                  │
│   1. VALIDATE ──► Prove your generation capacity                │
│         │                                                        │
│         ▼                                                        │
│   2. MINT ──────► Create eJLE (max 100 days of capacity)        │
│         │                                                        │
│         ▼                                                        │
│   3. SELL ──────► Trade on CEX/DEX or direct sales              │
│         │                                                        │
│         ▼                                                        │
│   4. BURN ──────► Accept eJLE for goods/services                │
│         │                                                        │
│         ▼                                                        │
│   5. MINT AGAIN ► Burning unlocks new minting capacity          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 Minting Rules

| Rule | Description |
|------|-------------|
| **Capacity Validation** | Must prove generation capacity via oracle/validator network |
| **Minting Cap** | Maximum 100 days of validated daily generation |
| **Burn-to-Mint** | Must burn tokens to unlock new minting capacity |
| **Revalidation** | Capacity must be periodically revalidated |

**Mintable Amount Formula:**

```
mintableAmount = min(
    dailyCapacity × 100,              // Max 100 days cap
    totalBurned - totalMinted + bootstrap   // Burn-to-mint balance
)
```

### 4.3 Supply Mechanics

```
Total Supply = Σ(Minted) - Σ(Burned)

Supply is limited by:
├── Validated capacity
├── 100-day cap per generator
└── Must burn to mint more

This creates natural equilibrium:
├── No unlimited inflation
├── Supply tied to real productive capacity
└── Active economy required to maintain supply
```

### 4.4 Price Discovery

eJLE price is **market-driven**, determined by supply and demand on DEX/CEX:

```
1 eJLE = 1 kWh (physical unit, always)
Price in USD = market rate (anchor ~$0.10)
```

**Anchor Mechanism:** Since eJLE is redeemable for real energy/goods/services, price naturally gravitates toward average global energy cost (~$0.10/kWh).

### 4.5 Fee Structure

**Transaction Fee: 0.1%**

Every eJLE transfer incurs a 0.1% fee distributed as follows:

| Recipient | Share | Purpose |
|-----------|-------|---------|
| Staking Pool | 70% | Staker rewards |
| DAO Treasury | 20% | Development, grants |
| Insurance Fund | 10% | Dispute payouts, emergencies |

**Example:**
```
Transfer: 10,000 eJLE

├── Stakers: 7 eJLE
├── Treasury: 2 eJLE
└── Insurance: 1 eJLE
```

## 5. Staking & Sponsorship Model

### 5.1 100% Collateral Backing

Every eJLE in circulation is backed by staked crypto assets. This ensures protocol solvency and prevents token dumping.

### 5.2 Sponsored Minting

```
┌─────────────────────────────────────────────────────────────────┐
│  SPONSORED MINTING MODEL                                        │
│                                                                  │
│  STAKER (has capital):                                          │
│  ├── Stakes ETH/MATIC/USDC as collateral                        │
│  ├── Opens minting slots for newcomers                          │
│  └── Earns share of 0.1% transaction fees                       │
│                                                                  │
│  NEWCOMER (from waitlist):                                      │
│  ├── Completes BrightID verification                            │
│  ├── Joins waitlist                                             │
│  ├── Gets sponsored by a staker                                 │
│  └── Receives minting rights without own collateral             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 Staking Mechanics

| Stake Amount | Opens Slots For | Collateral Ratio |
|--------------|-----------------|------------------|
| $1,000 | 10,000 eJLE | 100% |
| $10,000 | 100,000 eJLE | 100% |
| $100,000 | 1,000,000 eJLE | 100% |

### 5.4 Staker Rewards

Stakers earn from the 70% of transaction fees allocated to the staking pool:

```
Example:
Total Staking Pool: $10,000,000 TVL
Your Stake: $10,000 (0.1% of pool)

Daily Network Volume: 10,000,000 eJLE
Daily Fees: 10,000 eJLE (0.1%)
Staker Pool Share: 7,000 eJLE (70%)
Your Daily Reward: 7 eJLE (~$0.70)

Monthly: ~210 eJLE (~$21)
Yearly: ~2,555 eJLE (~$255)
APY: ~2.55% (depends on network activity)
```

### 5.5 Unstaking

- **Cooldown Period**: 14 days
- **Early Exit Penalty**: 5% if unstaking before cooldown
- **Partial Unstaking**: Allowed if remaining stake covers sponsored slots

## 9. Arbitration System

### 9.1 Overview

Joule Bond implements a **4-tier arbitration system** combining AI, crowdsourced jurors, and professional arbitrators for comprehensive dispute resolution.

```
┌─────────────────────────────────────────────────────────────────┐
│              JOULE BOND ARBITRATION SYSTEM                       │
│                                                                  │
│  TIER 0: <10,000 eJLE                                           │
│  └── AI Arbitrator (decentralized compute)                      │
│                                                                  │
│  TIER 1: 10,000 — 100,000 eJLE                                  │
│  └── Kleros (3 jurors) + AI Review                              │
│                                                                  │
│  TIER 2: 100,000 — 1,000,000 eJLE                               │
│  └── Professional (1) + Kleros (3) + AI Review                  │
│                                                                  │
│  TIER 3: >1,000,000 eJLE                                        │
│  └── Professional Panel (3) + AI Review                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 9.2 Dispute Types

| Category | Examples |
|----------|----------|
| **Delivery** | Non-delivery, partial delivery, quality issues |
| **Energy** | Grid non-delivery, metering disputes |
| **Capacity** | False capacity claims, validator errors |
| **Identity** | Sybil attacks, identity theft, false KYB |
| **Sponsorship** | Sponsor fraud, generator abandonment |

### 9.3 Arbitration Fee

**Flat 3% Fee** — Paid by the losing party

```
Fee Distribution:
├── 60% → Arbitrators (split equally)
├── 30% → Insurance Fund
└── 10% → DAO Treasury
```

**Insolvency Handling:**
- If loser cannot pay → debt recorded on-chain
- Future mints automatically repay debt first
- Debt has priority over all other operations

### 9.4 Tier 0: AI Arbitration (<10,000 eJLE)

```
┌─────────────────────────────────────────────────────────────────┐
│            TIER 0: AI ARBITRATION                                │
│                                                                  │
│  Provider: Decentralized AI compute (e.g., Gonka.ai)            │
│                                                                  │
│  Process:                                                       │
│  1. Dispute opened, both parties submit evidence                │
│  2. 3 compute nodes run AI model independently                  │
│  3. Consensus: 2/3 must agree on verdict                        │
│  4. Verdict + reasoning published                               │
│  5. 24h challenge period                                        │
│                                                                  │
│  Time: ~30 minutes                                              │
│  Cost: ~$1-5 compute + 3% fee                                   │
│  Appeal: → Tier 1 (appellant pays Kleros fee)                   │
│                                                                  │
│  AI Model:                                                      │
│  ├── Fine-tuned LLM for Joule Bond disputes                     │
│  ├── Training: historical cases, energy contracts               │
│  ├── Output: verdict (0-100%) + confidence + reasoning          │
│  └── Weights stored on IPFS/Arweave                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 9.5 AI Review System

For Tiers 1-3, AI provides a "second opinion" on human arbitrator decisions:

```
AI Review Output:
{
  "ai_verdict": 70,              // AI's independent verdict
  "arbitrator_verdict": 80,      // Human arbitrators' decision
  "deviation": 10,               // Difference
  "fairness_confidence": "HIGH", // HIGH / MEDIUM / LOW
  "appeal_eligible": false,      // true if LOW confidence
  "concerns": []                 // Specific issues identified
}

Confidence Levels:
├── HIGH (<15% deviation): Verdict final, no appeal
├── MEDIUM (15-30%): Verdict stands, appeal possible (full fee)
└── LOW (>30%): Appeal encouraged (50% fee discount)
```

### 9.6 Tier Details

| Tier | Amount | Primary | Review | Time | Appeal |
|------|--------|---------|--------|------|--------|
| **0** | <10k | AI (3 nodes) | — | 30 min | → Tier 1 |
| **1** | 10k-100k | Kleros (3) | AI | 7-21 days | → Tier 2 (if AI LOW) |
| **2** | 100k-1M | Professional (1) | Kleros + AI | 30-60 days | → Tier 3 (if disagree) |
| **3** | >1M | Panel (3) | AI | 60-90 days | → ICC/LCIA |

### 9.7 Professional Arbitrators

**Requirements:**
- Certified arbitrator (JAMS/AAA/ICC/LCIA) OR
- Licensed attorney + 3yr dispute resolution OR
- Energy industry executive + mediation training

**Onboarding:**
1. Application + credentials
2. KYB verification
3. DAO vote (simple majority)
4. Stake 50,000 eJLE
5. Sign NDA + Code of Ethics

### 9.8 Confidentiality

| Tier | Method |
|------|--------|
| 0-1 | Lit Protocol encryption (arbitrators only can decrypt) |
| 2 | Lit Protocol + NDA |
| 3 | Full NDA + legal liability |

Evidence is stored encrypted on IPFS; only assigned arbitrators receive decryption access.

## 12. Security Considerations

### 12.1 Attack Vectors & Mitigations

| Attack | Mitigation |
|--------|------------|

| **Oracle Manipulation** | 3/5 validator consensus, stake slashing |

### 12.2 Security Measures

- **No Admin Keys**: All control via DAO governance
- **Timelock**: 48h-14d delays on all governance actions
- **Multi-sig Oracle**: 3/5 consensus for capacity validation
- **Reentrancy Guards**: All external calls protected
- **Pausability**: Emergency stop for critical contracts

## 13. Legal & Compliance

### 13.1 Token Classification

eJLE is designed as a **utility token**:

- Represents productive capacity, not investment returns
- No profit-sharing or dividend mechanisms
- Value derived from utility (energy redemption)
- Burn mechanism for consumption

### 13.2 Jurisdictional Considerations

- Protocol is permissionless and decentralized
- Individual compliance is user responsibility
- KYB verification available for regulated entities
- Arbitration verdicts may have limited enforceability in some jurisdictions

### 13.3 Privacy Compliance

- Default pseudonymous operation
- KYB data stored off-chain (hash only on-chain)
- GDPR-compatible: users control their data
- Right to deletion: off-chain data can be removed
